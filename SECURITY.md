# Security Policy

PhishGuard AI is an educational portfolio project.

## Data handling

The application processes uploaded `.eml` files in the browser. It does not intentionally send email contents to any backend or third-party API.

## Important limitations

- Do not use this as the only control for phishing detection.
- Do not upload confidential emails to public environments unless your organization allows it.
- Do not paste sensitive email content into public AI tools.
- Static GitHub Pages apps cannot safely store private API keys.

## Reporting issues

If you find a bug, open an issue in the GitHub repository with:

- clear reproduction steps;
- browser and operating system;
- whether the issue affects parsing, scoring or export.
